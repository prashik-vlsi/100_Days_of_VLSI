# Day 30 — AHB-Lite Slave

**Project:** NeuralEdge SoC — Edge AI Inference Accelerator
**Engineer:** Prashik Wankhede
**Milestone:** AHB Master + Slave — complete bus system

## Overview

Day 30 completes the AHB-Lite memory subsystem for NeuralEdge by building the
**AHB-Lite Slave** and integrating it with the Day 29 Master into a single,
verified Master–Slave system. This is the memory backbone that will hold
neural network weights for AI inference.

## What Was Built

### AHB-Lite Slave (`ahb_slave.v`)

- Standard AHB-Lite slave port interface (HSELx, HREADY, HRESP, etc.)
- Address phase capture (`addr_reg`, `write_reg`) — registers HADDR/HWRITE
  during the address phase so the correct values are used in the data phase,
  even though AHB is pipelined and the Master has already moved on.
- Internal memory array — 1024 x 32-bit (`mem_array[0:1023]`), indexed using
  `addr_reg[11:2]` (drops the 2 always-zero word-alignment bits).
- HREADYOUT generation with a 2-cycle wait-state counter to simulate a slow
  memory.
- HRESP error response — asserted when `addr_reg[31:12] != 0`, i.e. the
  address falls outside the 1024-location valid range.
- HRDATA — reads from `mem_array` using the captured address, gated to avoid
  driving during write transactions.

### AHB-Lite Master (`ahb_master.v`) — upgraded from Day 29

- Added `burst_mode` input — selects between SINGLE and INCR4 burst transfers.
- Added `data_in` input — lets the testbench supply real write data instead
  of a hardcoded constant.
- Added pipelined read-data capture (`valid_rphase` flag + `rdata_out`
  register) so the Master can actually capture and report back data read
  from the Slave.

### Testbench (`ahb_system_tb.v`)

- Instantiates Master and Slave, connected via shared AHB signals
  (HADDR, HWRITE, HTRANS, HWDATA, HRDATA, HRESP, HREADY/HREADYOUT loopback).
- `do_transfer` task to drive single/burst read & write transactions.
- Test sequence:
  1. Single write of 42 to address `0x1000`
  2. Single read-back from `0x1000`, verified against expected value
  3. INCR4 burst write
  4. INCR4 burst read
  5. Invalid address access — verifies HRESP error response

## Key Concepts

- **Address phase capture for pipelining** — AHB's pipelined address/data
  phases mean a slave must register HADDR, or it will use the _next_
  transfer's address by mistake.
- **HREADY generation, slave side** — wait states are inserted by holding
  HREADYOUT low for a fixed number of cycles via a down-counter.
- **HRESP error response** — generated by checking the upper address bits
  against the slave's valid memory range.
- **Pipelined read-data capture in the Master** — a one-cycle-delayed
  "valid read phase" flag is needed to correctly capture HRDATA, since the
  data isn't valid until one cycle after the address phase.
