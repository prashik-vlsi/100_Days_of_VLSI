# Day 25 — I2C Master

**Topic:** I2C Master — Start/Stop Condition Generation, ACK/NACK Handling, 7-bit Addressing, Data Phase Read/Write, Clock Stretching, and AD8232 ECG Sensor Interface Verification.

---

## Theory

The I2C Master initiates and controls all transactions on the I2C bus. It generates the SCL clock, drives SDA for Start and Stop conditions, transmits the 7-bit slave address with a R/W bit, handles ACK/NACK responses from the slave, and manages data phase transfers in both directions.

In the **VitalGuard ECG Signal Processor SoC**, the AD8232 ECG front-end analog IC is read over I2C. The Master polls the AD8232 registers to extract real-time ECG waveform samples for downstream digital processing. In the **ShieldHer Women Safety Alert SoC**, the MPU6050 accelerometer is interfaced via I2C — the Master reads acceleration registers to detect distress motion events and trigger the GSM SOS pipeline.

## Key Concepts

### I2C Signal Directions (Master Perspective)

| Signal | Direction              | Purpose                                      |
| ------ | ---------------------- | -------------------------------------------- |
| SCL    | Output (Master drives) | Clock generated by Master                    |
| SDA    | Bidirectional          | Master drives address/data; Slave drives ACK |

### Start and Stop Conditions

| Condition      | Definition                                                |
| -------------- | --------------------------------------------------------- |
| START          | SDA falls LOW while SCL is HIGH                           |
| STOP           | SDA rises HIGH while SCL is HIGH                          |
| Repeated START | START issued without preceding STOP — keeps bus ownership |

```
START condition:
SCL  ──────────────┐
                   └──────
SDA  ─────┐
          └───────────────

STOP condition:
SCL        ┌──────────────
      ─────┘
SDA              ┌────────
      ───────────┘
```

### 7-bit Address Frame

```
[ START ] [ A6 A5 A4 A3 A2 A1 A0 ] [ R/W ] [ ACK ] [ DATA ... ] [ STOP ]
            ───── 7-bit address ─────   0=W
                                        1=R
```

- Master transmits 8 bits: 7 address bits + 1 R/W bit
- Slave pulls SDA LOW on the 9th SCL pulse = ACK
- SDA remains HIGH on 9th SCL pulse = NACK

### ACK/NACK Handling

| Condition | SDA on 9th clock       | Meaning                                 |
| --------- | ---------------------- | --------------------------------------- |
| ACK       | LOW (slave drives)     | Slave received byte, ready for next     |
| NACK      | HIGH (not pulled down) | Slave busy, error, or last byte of read |

### Clock Stretching

The slave holds SCL LOW to pause the Master when it needs more processing time. The Master must detect SCL held LOW and wait before continuing. This is the only case where the Master loses SCL control.

```
SCL  ───┐  ┌───┐        ┌───   ← Master releases SCL
        └──┘   └────────┘
                ↑ Slave holds SCL LOW (stretching)
```

```
          │  START/STOP generator                   ├──► sda (bidir)

stop ────►│  FSM controller                         │
          └─────────────────────────────────────────┘
```

## FSM State Diagram

```
IDLE ──start──► START_COND ──► ADDR_PHASE ──► ACK1
                                                │
                              ┌─── ACK ◄────────┘
                              │
                         DATA_PHASE ──► ACK2
                              │
                         STOP_COND ──► IDLE

                         (NACK at any ACK state → STOP + ack_error)
```

### State Descriptions

| State      | Action                                           |
| ---------- | ------------------------------------------------ |
| IDLE       | Bus free, awaiting start command                 |
| START_COND | Pull SDA LOW while SCL HIGH                      |
| ADDR_PHASE | Shift out 7-bit address + R/W bit, 8 SCL pulses  |
| ACK1       | Release SDA, sample slave ACK on 9th SCL pulse   |
| DATA_PHASE | Shift out (write) or shift in (read) 8 data bits |
| ACK2       | Sample ACK after data byte                       |
| STOP_COND  | Release SDA HIGH while SCL HIGH                  |

## I2C Transaction Timing

```
SCL  ─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌─┐ ┌────
      └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘ └─┘

SDA  ─┐[A6][A5][A4][A3][A2][A1][A0][RW][ACK][D7][D6][D5][D4][D3][D2][D1][D0][ACK]─┐
      S                                                                              P
   START                                                                           STOP

S = START: SDA falls while SCL HIGH
P = STOP:  SDA rises while SCL HIGH
ACK = slave pulls SDA LOW on 9th clock
```

## Functional Blocks

### START Condition Generator

- **Purpose:** Initiate I2C transaction
- **Implementation:** Pull SDA LOW while SCL is HIGH, then pull SCL LOW
- **Timing:** SDA must be stable LOW before SCL falls

### STOP Condition Generator

- **Purpose:** Release the I2C bus
- **Implementation:** Pull SCL HIGH, then release SDA HIGH
- **Timing:** SCL must be HIGH before SDA rises

### Address + R/W Shift Register

- **Purpose:** Serialize 7-bit address and R/W direction bit
- **Implementation:** `shift_reg <= {shift_reg[6:0], 1'b0}` — MSB first
- **Length:** 8 bits total (addr[6:0] + rw)

### ACK Sampler

- **Purpose:** Read slave acknowledgement on 9th SCL pulse
- **Implementation:** Sample SDA during SCL HIGH on 9th clock
- **Action:** If SDA == 1 (NACK) → assert ack_error, go to STOP

### Clock Stretch Detector

- **Purpose:** Detect when slave holds SCL LOW
- **Implementation:** After Master releases SCL, poll SCL pin — wait until HIGH
- **Impact:** Master must not proceed until SCL returns HIGH

### Data TX Shift Register (Write)

- **Purpose:** Serialize tx_data[7:0] onto SDA
- **Implementation:** `sda_out <= tx_shift[7]; tx_shift <= {tx_shift[6:0], 1'b0}`

### Data RX Shift Register (Read)

- **Purpose:** Deserialize 8 SDA bits into rx_data
- **Implementation:** `rx_shift <= {rx_shift[6:0], sda_in}`
- **Latch:** `rx_data <= {rx_shift[6:0], sda_in}` when bit_cnt == 7

## Key Learnings

### START and STOP Are Not Data Bits

START and STOP are defined by SDA transitions **while SCL is HIGH** — the only time SDA is allowed to change. During data and address phases, SDA must be stable while SCL is HIGH and can only change while SCL is LOW. Violating this rule creates a false START or STOP mid-transaction.

### ACK is Slave-Driven, NACK is Passive

The Master releases SDA (tri-state or HIGH) during the 9th clock pulse. The slave actively drives SDA LOW to ACK. If the slave does not respond, SDA stays HIGH by the pull-up resistor — that is NACK. The Master must sample SDA while SCL is HIGH on the 9th pulse.

### Clock Stretching Requires SCL to Be Bidirectional

SCL is an output from the Master, but the slave can hold it LOW. This requires SCL to be implemented as open-drain with a pull-up — the Master must read back SCL after releasing it to confirm the slave is not stretching. If SCL reads LOW after release, the Master must wait.

### Why I2C Uses Open-Drain + Pull-Up

Both SDA and SCL are open-drain lines. Any device on the bus can pull them LOW but no device actively drives them HIGH — the pull-up resistor does that. This allows multiple masters (multi-master bus) and clock stretching by slaves without bus contention.

## Capstone Connections

### VitalGuard — ECG Signal Processor SoC

- **Sensor:** AD8232 ECG front-end (I2C address: 0x1A)
- **Interface:** This I2C Master reads ECG output registers every sampling period
- **Data flow:** rx_data → ECG digital filter → QRS detector → heart rate computation
- **Critical timing:** I2C read must complete before next ECG sample window opens

### ShieldHer — Women Safety Alert SoC

- **Sensor:** MPU6050 accelerometer (I2C address: 0x68)
- **Interface:** Master polls acceleration registers at 100Hz
- **Data flow:** rx_data → motion classifier → distress detection → GSM SOS trigger
- **Critical timing:** ACK failure on MPU6050 must trigger sensor fault alert, not silent failure
