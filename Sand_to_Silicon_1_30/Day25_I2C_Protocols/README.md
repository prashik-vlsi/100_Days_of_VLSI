Day 25 вҖ” I2C Master

Topic: I2C Master вҖ” Start/Stop Condition Generation, ACK/NACK Handling, 7-bit Addressing, Data Phase Read/Write, Clock Stretching, and AD8232 ECG Sensor Interface Verification.

Theory

The I2C Master initiates and controls all transactions on the I2C bus. It generates the SCL clock, drives SDA for Start and Stop conditions, transmits the 7-bit slave address with a R/W bit, handles ACK/NACK responses from the slave, and manages data phase transfers in both directions.

In the VitalGuard ECG Signal Processor SoC, the AD8232 ECG front-end analog IC is read over I2C. The Master polls the AD8232 registers to extract real-time ECG waveform samples for downstream digital processing. In the ShieldHer Women Safety Alert SoC, the MPU6050 accelerometer is interfaced via I2C вҖ” the Master reads acceleration registers to detect distress motion events and trigger the GSM SOS pipeline.

Functional Blocks

START Condition Generator

Purpose: Initiate I2C transaction
Implementation: Pull SDA LOW while SCL is HIGH, then pull SCL LOW
Timing: SDA must be stable LOW before SCL falls

STOP Condition Generator

Purpose: Release the I2C bus
Implementation: Pull SCL HIGH, then release SDA HIGH
Timing: SCL must be HIGH before SDA rises

Address + R/W Shift Register

Purpose: Serialize 7-bit address and R/W direction bit
Implementation: shift_reg <= {shift_reg[6:0], 1'b0} вҖ” MSB first
Length: 8 bits total (addr[6:0] + rw)

ACK Sampler

Purpose: Read slave acknowledgement on 9th SCL pulse
Implementation: Sample SDA during SCL HIGH on 9th clock
Action: If SDA == 1 (NACK) вҶ’ assert ack_error, go to STOP

Clock Stretch Detector

Purpose: Detect when slave holds SCL LOW
Implementation: After Master releases SCL, poll SCL pin вҖ” wait until HIGH
Impact: Master must not proceed until SCL returns HIGH

Data TX Shift Register (Write)

Purpose: Serialize tx_data[7:0] onto SDA
Implementation: sda_out <= tx_shift[7]; tx_shift <= {tx_shift[6:0], 1'b0}

Data RX Shift Register (Read)

Purpose: Deserialize 8 SDA bits into rx_data
Implementation: rx_shift <= {rx_shift[6:0], sda_in}
Latch: rx_data <= {rx_shift[6:0], sda_in} when bit_cnt == 7

Key Learnings

START and STOP Are Not Data Bits

START and STOP are defined by SDA transitions while SCL is HIGH вҖ” the only time SDA is allowed to change. During data and address phases, SDA must be stable while SCL is HIGH and can only change while SCL is LOW. Violating this rule creates a false START or STOP mid-transaction.

ACK is Slave-Driven, NACK is Passive

The Master releases SDA (tri-state or HIGH) during the 9th clock pulse. The slave actively drives SDA LOW to ACK. If the slave does not respond, SDA stays HIGH by the pull-up resistor вҖ” that is NACK. The Master must sample SDA while SCL is HIGH on the 9th pulse.

Clock Stretching Requires SCL to Be Bidirectional

SCL is an output from the Master, but the slave can hold it LOW. This requires SCL to be implemented as open-drain with a pull-up вҖ” the Master must read back SCL after releasing it to confirm the slave is not stretching. If SCL reads LOW after release, the Master must wait.

Why I2C Uses Open-Drain + Pull-Up

Both SDA and SCL are open-drain lines. Any device on the bus can pull them LOW but no device actively drives them HIGH вҖ” the pull-up resistor does that. This allows multiple masters (multi-master bus) and clock stretching by slaves without bus contention.
